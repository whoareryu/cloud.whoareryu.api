# 타이타닉 데이터셋 설명서 (Titanic Dataset Description)

이 문서는 타이타닉 생존자 예측 모델을 구축하기 위한 데이터셋 구조와 각 변수(Feature)의 명세 및 피처 엔지니어링(Feature Engineering) 인사이트를 정리한 가이드라인입니다.

---

## 1. 데이터셋 개요 (Dataset Overview)

데이터는 크게 두 개의 그룹으로 나누어져 있습니다.

1. **학습 데이터셋 (`train.csv`)**
   - 머신러닝 모델을 학습시키는 데 사용됩니다.
   - 각 승객의 생존 여부 결과(Ground Truth, 정답 라벨)가 포함되어 있습니다.
   - 승객의 성별, 티켓 등급 등의 피처(Features)를 기반으로 모델을 학습시키며, 기존 피처들을 조합하여 새로운 피처를 만들어내는 **피처 엔지니어링**을 수행할 수 있습니다.

2. **테스트 데이터셋 (`test.csv`)**
   - 학습된 모델이 보지 못한 새로운 데이터에 대해 얼마나 잘 예측하는지 성능을 검증하는 데 사용됩니다.
   - 생존 여부(정답 라벨)가 포함되어 있지 않습니다. 모델을 사용하여 각 승객의 생존 여부를 예측해야 합니다.

3. **제출 샘플 파일 (`gender_submission.csv`)**
   - "여성 승객은 모두 생존하고, 남성 승객은 모두 사망했다"고 가정한 단순 예측 모델의 결과 예시입니다. 최종 제출 파일의 형태(포맷)를 참고하는 용도로 사용됩니다.

## 2. 데이터 사전 (Data Dictionary)

| 변수명 (Variable) | 정의 (Definition) | 값 설명 및 범주 (Key) |
| :--- | :--- | :--- |
| **survival** | 생존 여부 (타겟 변수) | 0 = 사망 (No), 1 = 생존 (Yes) |
| **pclass** | 티켓 등급 (사회·경제적 지위) | 1 = 1등석 (상류층), 2 = 2등석 (중산층), 3 = 3등석 (하류층) |
| **sex** | 성별 | male = 남성, female = 여성 |
| **age** | 나이 (세 단위) | 1세 미만은 소수점으로 표시, 추정 나이는 xx.5 형태로 표시 |
| **sibsp** | 함께 탑승한 형제자매 / 배우자 수 | 형제, 자매, 의붓형제, 의붓자매, 남편, 아내 |
| **parch** | 함께 탑승한 부모 / 자녀 수 | 어머니, 아버지, 딸, 아들, 의붓딸, 의붓아들 (보모와 동행한 아이는 0) |
| **ticket** | 티켓 번호 | 문자열 또는 숫자 조합의 티켓 식별자 |
| **fare** | 승객 요금 | 탑승을 위해 지불한 비용 |
| **cabin** | 객실 번호 | 승객이 머문 객실 구역 및 방 번호 |
| **embarked** | 탑승 항구 (Port of Embarkation) | C = 셰르부르(Cherbourg), Q = 퀸즈타운(Queenstown), S = 사우샘프턴(Southampton) |

## 3. 피처 세부 참고사항 (Variable Notes)

* **pclass (티켓 등급):** 승객의 사회적·경제적 지위(SES, Socio-Economic Status)를 나타내는 대리 변수(Proxy) 역할을 합니다.
* **age (나이):** 1세 미만의 영유아 데이터는 분수/소수점으로 표현됩니다. 나이가 정확하지 않고 추정된 경우 `xx.5` 형식으로 기록되어 있습니다.
* **sibsp (가족 관계 - 동등 계층):**
  - 형제자매(Sibling) = 남동생, 여동생, 형, 누나, 오빠, 의붓형제, 의붓자매
  - 배우자(Spouse) = 남편, 아내 (단, 약혼자 및 내연 관계는 제외됨)
* **parch (가족 관계 - 세대 계층):**
  - 부모(Parent) = 어머니, 아버지
  - 자녀(Child) = 딸, 아들, 의붓딸, 의붓아들
  - *참고:* 부모 없이 보모(Nanny)와 함께 여행한 일부 아이들의 경우 `parch = 0`으로 기록되었습니다.

## 4. 피처 엔지니어링을 위한 분석 및 아이디어 기획 (Feature Engineering Insights)

이 데이터셋을 활용해 AI 모델이 더 높은 예측력을 가질 수 있도록 유도할 수 있는 대표적인 피처 엔지니어링 전략 가이드라인입니다.

### ① 가족 규모 통합 피처 (Family Size)
* **아이디어:** `sibsp`(형제자매/배우자)와 `parch`(부모/자녀)는 모두 승객이 혼자가 아니라 '가족과 함께 탑승했는가'를 나타냅니다. 이 둘을 합쳐 새로운 변수를 만듭니다.
* **수식:** `FamilySize = sibsp + parch + 1` (본인 포함)
* **추가 세분화:** 1명(혼자 탑승 - `IsAlone`), 2~4명(소형 가족), 5명 이상(대형 가족)으로 범주화(Binning)할 수 있습니다. 대가족일수록 대피 통제가 어려워 생존율이 낮아지는 경향을 모델에 학습시킬 수 있습니다.

### ② 이름(Name)에서 사회적 지위 및 성별 추출 (Title Extraction)
* **아이디어:** 승객의 이름 데이터는 `Braund, Mr. Owen Harris` 또는 `Heikkinen, Miss. Laina` 형태로 제공됩니다. 이름 중간에 위치한 호칭(Title)인 `Mr.`, `Mrs.`, `Miss.`, `Master.`, `Dr.`, `Capt.` 등을 추출할 수 있습니다.
* **효과:** `Master`는 어린 소년을 의미하므로 나이(`age`)의 결측치(NaN)를 추정할 때 유용하며, `Dr.`, `Countess`(백작부인), `Sir` 등은 높은 사회적 지위를 나타내어 생존율 예측에 중요한 단서가 됩니다.

### ③ 나이(Age)의 결측치 처리 및 범주화 (Age Binning)
* **아이디어:** 나이는 결측치(Missing Value)가 많습니다. 단순히 전체 평균으로 채우기보다는 앞서 추출한 **호칭(Title)별 나이 평균** 또는 **pclass별 나이 평균**을 구해 정교하게 결측치를 대체(Imputation)합니다.
* **범주화:** 나이를 연속형 변수에서 영유아(Infant), 아동(Child), 청소년(Teenager), 성인(Adult), 노년층(Senior)과 같은 범주형 데이터로 변환하여 모델의 과적합(Overfitting)을 방지합니다. ("여성과 아이를 먼저 구한다"는 재난 당시의 구조 원칙 반영 가능)

### ④ 객실(Cabin) 정보를 통한 구역 파악 (Deck Isolation)
* **아이디어:** 객실 번호(`cabin`)는 대부분 결측치이지만, 값이 있는 경우 `C23`, `E46`과 같이 알파벳으로 시작합니다. 이 첫 글자는 타이타닉호의 **구역(Deck, 갑판)**을 의미합니다.
* **효과:** 구역(A~G)에 따라 구명보트가 있는 상부 갑판과의 거리 및 접근성이 다르므로 생존율에 직접적인 영향을 미칩니다. 값이 없는 데이터는 `Unknown`으로 처리하여 하나의 범주로 활용할 수 있습니다.

### ⑤ 티켓(Ticket)과 요금(Fare)의 재해석
* **아이디어:** 동일한 티켓 번호를 가진 승객들은 가족이나 동행인일 가능성이 높으며, 요금(`fare`) 역시 티켓 한 장에 여러 명의 요금이 합산되어 기록된 경우가 있습니다.
* **효과:** 티켓 번호별 빈도수를 계산하여 '동행자 수'를 파악할 수 있으며, 전체 요금을 동행자 수로 나누어 **1인당 실제 요금(Single Fare)**을 계산하면 승객의 정확한 경제적 수준을 더 정확하게 반영할 수 있습니다.
