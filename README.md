# com.ragwatson

이 저장소는 여러 하위 프로젝트와 문서를 담는 **워크스페이스**이며, AI 코딩 도구가 맥락을 잃지 않도록 **하네스 엔지니어링**(규약으로 입력·출력을 제한하는 설계)을 루트에 둔다.

에이전트 행동의 철학은 [Andrej Karpathy](https://x.com/karpathy)가 널리 제안한 **네 가지 원칙**과 같다: **구현 전 사고** → **단순성 우선** → **정밀한 수정** → **목표 중심 실행**(가정 금지·최소 변경·수술적 diff·검증 가능한 완료 기준).

> **트레이드오프:** 속도보다 **신중함**. 사소한 작업은 완화할 수 있으나, 침묵 가정·범위 팽창·검증 없는 “완료”는 피한다.

---

## 네 원칙 한 줄 요약

1. **Think Before Coding:** 가정은 말로 낸다. 불확실하면 질문한다. 해석이 갈리면 대안을 나열한다.
2. **Simplicity First:** 요청을 만족하는 최소 코드만 쓴다. 안 시킨 기능·추상화·방어 코드를 넣지 않는다.
3. **Surgical Changes:** 요청과 직결된 줄만 바꾼다. 옆줄 “정리”·무관 리팩터는 하지 않는다.
4. **Goal-Driven Execution:** 성공 조건을 먼저 정하고, 가능하면 테스트·동등 검증으로 끝을 맞춘다.
